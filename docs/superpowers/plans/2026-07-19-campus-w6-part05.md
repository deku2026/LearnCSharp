# Campus W6 (Part05_1–2) Implementation Plan

**Goal:** Real Keycloak-based authentication/authorization and a same-origin SPA/BFF security lab backed by Redis.

**Labs:** Part05_1_AuthnAuthz · Part05_2_SpaAuth

**Branch:** `codex/w6-part05-auth` · Worktree: `.worktrees/campus-w6-part05`

---

### Task 1: Part05_1 authentication and API authorization

- Keycloak OIDC/JWKS and confidential code + PKCE web client
- strict issuer/audience/lifetime/signing-key validation
- scope, role and owner-resource policies
- explicit CORS, per-user/IP rate limiting and Problem Details
- front-channel and back-channel logout

### Task 2: Shared server-side security state

- opaque-cookie `ITicketStore`
- Redis authentication tickets and revocation markers
- shared Redis Data Protection key ring for multiple instances

### Task 3: Part05_2 same-origin SPA/BFF

- confidential OIDC code + PKCE client with PAR when available
- no browser token storage; HttpOnly/SameSite session
- CSRF header plus Origin validation
- YARP downstream proxy with server-side token attachment
- refresh-token rotation, revocation and logout

### Task 4: Real infrastructure validation

- reusable Keycloak Realm template and local provisioning script
- generic WebApplicationFactory tests on all operating systems
- real Keycloak/Redis/Kestrel tests on Linux CI only
- local Windows `.NET` tests may connect to the WSL Docker stack
